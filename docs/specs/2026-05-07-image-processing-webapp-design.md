# 图像处理 Web 应用设计规格

**日期：** 2026-05-07
**框架：** Streamlit
**数据集：** MNIST（主），Fashion-MNIST（可选切换）
**GPU：** NVIDIA RTX 5060 Laptop (8GB VRAM)

---

## 项目结构

```
image7/
├── app.py                    # Streamlit 入口，Tab 路由
├── models/
│   ├── autoencoder.py        # AE 模型定义与训练
│   ├── vae.py                # VAE 模型定义与训练
│   ├── dcgan.py              # DCGAN 生成器/判别器与训练
│   └── diffusion_loader.py   # SD 1.5 加载与推理
├── ui/
│   ├── tab_ae_vae.py         # Tab 1: AE vs VAE 对比
│   ├── tab_dcgan.py          # Tab 2: DCGAN 生成
│   └── tab_diffusion.py      # Tab 3: 文本到图像
├── utils/
│   ├── data_loader.py        # MNIST/Fashion-MNIST 数据加载
│   ├── visualization.py      # 热力图、散点图、loss 曲线
│   └── session_manager.py    # 显存管理，模型缓存
├── checkpoints/              # 预训练权重缓存 (.pt)
├── assets/                   # 静态生成样本
├── docs/
│   └── specs/                # 设计文档
└── requirements.txt
```

### 核心设计决策

1. **显存管理（LRU 缓存）：** 同一时间只加载一组模型。切换 Tab 时通过 `torch.cuda.empty_cache()` 卸载上一个 Tab 的模型。SD 1.5（~4GB）在 Tab 3 享有最高加载优先级。

2. **模型训练：** 首次进入对应 Tab 时触发训练（MNIST AE/VAE 约 1-2 分钟，DCGAN 约 3-5 分钟）。权重缓存到 `checkpoints/` 目录，后续进入直接加载缓存。

3. **状态管理：** 所有 UI 状态存在 `st.session_state`。每个 Tab 使用独立的 session key 前缀隔离（如 `ae_vae_*`、`dcgan_*`、`diffusion_*`）。

## Tab 1：自编码器与 VAE 重构对比

### 界面布局

```
┌──────────────────────────────────────────────────┐
│ 数据集: [MNIST ▼]  潜空间维度: [2 ▸ 20]           │
│ [训练模型]  [加载已有权重]                         │
├────────────────┬──────────────┬──────────────────┤
│ 原始输入        │ AE 重构       │ VAE 重构          │
│ [5×5 图像网格]  │ [5×5 图像网格]│ [5×5 图像网格]    │
├────────────────┴──────────────┴──────────────────┤
│ 重构误差热力图 (|原图 - 重构|)                    │
│ [AE 误差行] [VAE 误差行]  (各 5×5 张)             │
├──────────────────────────────────────────────────┤
│ Loss 曲线 (Plotly 交互)                          │
│ [AE 训练/验证 loss | VAE 总 loss/KL loss/重构 loss]│
├──────────────────────────┬───────────────────────┤
│ 潜空间散点图              │ 潜空间插值              │
│ [2D 散点图，按类别着色]   │ 样本 A: [选择]         │
│ [点击位置 → 解码图像]     │ 样本 B: [选择]         │
│                          │ [8 帧插值序列展示]      │
└──────────────────────────┴───────────────────────┘
```

### 功能组件

- **模型训练：** AE 可配置潜空间维度（2-20），VAE 同步。encoder/decoder 各 2-3 层卷积，约 20 万参数量。
- **重构对比网格：** 25 张随机测试样本，三列并排（原图、AE 重构、VAE 重构）。
- **误差热力图：** 逐像素绝对差值 `|x - x_hat|`，matplotlib 渲染热力图。
- **Loss 曲线：** Plotly 折线图，x 轴为 epoch，y 轴为 loss。分轨展示 AE 重构 loss、VAE 总 loss、KL loss、重构 loss。
- **潜空间散点：** 仅 2D（此面板强制 latent_dim=2）。Plotly 散点图按类别着色。点击事件 → decoder 在对应潜空间坐标生成图像。
- **插值：** 用户从散点图选两个样本，潜空间内 8 步线性插值，逐帧解码展示。

### 训练配置

- 训练轮数：20
- 批大小：128
- 优化器：Adam，学习率 1e-3
- VAE beta（KL 权重）：0.1

## Tab 2：DCGAN 生成

### 界面布局

```
┌──────────────────────────────────────────────────┐
│ 噪声维度: [100]  网格大小: [8×8]   [生成新样本]    │
│ [保存当前网格作为参考]                             │
├───────────────────────┬──────────────────────────┤
│ 生成样本网格           │ 判别器分数                │
│ [8×8 图像网格]        │ [柱状图: D(G(z))          │
│                       │  每样本分数，降序排列]     │
├───────────────────────┴──────────────────────────┤
│ 噪声可视化                                        │
│ [点击任意生成图像 → 展示其 100 维噪声向量          │
│  的热力图 (10×10 重塑)]                           │
├──────────────────────────────────────────────────┤
│ 训练进度对比                                      │
│ [滑块: epoch 1→50] [展示该 epoch 的生成网格]      │
│ [并排对比: epoch X vs epoch Y 生成质量]            │
└──────────────────────────────────────────────────┘
```

### 功能组件

- **生成器：** DCGAN 架构，100 维噪声 → 28×28×1 输出。3 层转置卷积。
- **判别器：** 3 层卷积 → sigmoid 输出。BCE loss 训练。
- **网格展示：** `torchvision.utils.make_grid` 渲染为 PIL 图像。
- **分数直方图：** Plotly 柱状图，展示当前批次每张图的 D(G(z))。
- **噪声热力图：** 点击生成图像 → 将其 100 维噪声重塑为 10×10 热力图。
- **Epoch 滑块：** 加载不同训练阶段的 checkpoint，展示对应生成质量。

### 训练配置

- 训练轮数：50
- 批大小：128
- 优化器：Adam，学习率 2e-4，betas=(0.5, 0.999)
- 噪声维度：100

## Tab 3：文本到图像（Diffusers）

### 界面布局

```
┌──────────────────────────────────────────────────┐
│ 侧边栏 (Streamlit sidebar)  │ 主区域              │
│                             │                     │
│ 正向提示词: [_____________] │ ┌────────┬────────┐ │
│ 负向提示词: [_____________] │ │ 结果1   │ 结果2   │ │
│                             │ │ [图像]  │ [图像]  │ │
│ 采样步数: [1-50, 默认20]    │ └────────┴────────┘ │
│ CFG 引导强度: [1.0-20.0]    │ ┌────────┬────────┐ │
│ 随机种子: [42] [🎲随机]     │ │ 结果3   │ 结果4   │ │
│                             │ │ [图像]  │ [图像]  │ │
│ 分辨率: [256 / 512]         │ └────────┴────────┘ │
│ [生成图像]                   │                     │
│                             │ 对比模式:           │
│ 已保存结果: 4/4            │ ○ 网格  ○ 并排       │
│ [清空全部]                  │ ○ 滑块分割           │
└──────────────────────────────────────────────────┘
```

### 功能组件

- **模型：** `runwayml/stable-diffusion-v1-5`，通过 diffusers 加载。FP16 精度，启用 attention slicing 适配 8GB 显存。
- **参数控制：** 正向提示词、负向提示词、采样步数、引导强度（CFG）、随机种子、分辨率（256 或 512）。
- **多轮对比：** 最多保存 4 组生成结果及其参数。2×2 网格展示。
- **并排对比模式：** 任选 2 个结果，左右并排展示，支持拖拽滑块分割线。
- **参数标注：** 每张生成图像下方展示对应参数（步数、CFG、种子）。

### 性能预估

- SD 1.5 @ 512×512，20 步：RTX 5060 约 8 秒
- SD 1.5 @ 256×256：约 3 秒
- 模型按 Tab 切换加载/卸载，由 session_manager 统一调度

## 数据流

```
用户操作 → st.session_state 更新 → 模型推理 → UI 刷新
                              ↑
                  session_manager 检查显存，
                  必要时卸载上一个 Tab 的模型
```

### Session State 键值表

| 键名 | Tab | 用途 |
|------|-----|------|
| `loaded_tab` | 全局 | 当前激活的 Tab，用于模型切换 |
| `ae_vae_trained` | Tab 1 | 模型是否已训练 |
| `ae_vae_latent_dim` | Tab 1 | 当前潜空间维度 |
| `ae_vae_dataset` | Tab 1 | 数据集选择 (mnist/fashion) |
| `dcgan_trained` | Tab 2 | 模型是否已训练 |
| `dcgan_noise_batch` | Tab 2 | 当前噪声张量 |
| `dcgan_ref_grid` | Tab 2 | 保存的参考网格 |
| `diffusion_runs` | Tab 3 | 已生成结果列表 [(参数, 图像), ...] |
| `diffusion_pipe_loaded` | Tab 3 | SD pipeline 是否已加载 |

## 错误处理

- **CUDA OOM：** 捕获异常，提示降低分辨率或 batch size，展示当前显存使用量。
- **Diffusers 下载失败：** 展示重试按钮与进度。模型缓存至 `~/.cache/huggingface/`。
- **训练超时：** Streamlit 长任务回调 + 进度条。超过 10 分钟提示减少 epoch。
- **模型未训练：** 推理控件置灰，突出显示"训练模型"按钮，直到训练完成。
